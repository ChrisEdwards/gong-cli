# gong-cli

A CLI that syncs Gong call recordings into a local folder of markdown transcript files. General-purpose and publishable, with one reference workflow (a daily morning-routine sync into a personal knowledge base) driving v1.

## Language

**Call**:
A single recorded Gong conversation, identified by a Gong call ID.
_Avoid_: Meeting, recording, conversation

**Call File**:
The rendered markdown document for one Call, named `YYYY-MM-DD - <title>.md`, containing frontmatter, a Summary, an Outline, and a Transcript.
_Avoid_: Export, note, document

**Outline**:
Gong's AI-generated segmentation of a Call into titled, timestamped sections. Rendered both as a TOC and as the Transcript's section headings.
_Avoid_: Topics (Gong's topic taxonomy is a different, coarser thing)

**Turn**:
One speaker's contiguous monologue in the Transcript, rendered as `M:SS | Speaker Name` followed by text.
_Avoid_: Monologue, utterance

**Account**:
The CRM account (customer organization) a Call belongs to, from Gong's CRM context.
_Avoid_: Customer (as a field name), company

**Transcript**:
The speaker-attributed, timestamped record of what was said on a Call, rendered as monologue turns.

**Spotlight Summary**:
Gong's AI-generated digest of a Call, made up of a Brief, Key Points, and Next Steps. Generated by Gong some time after the Call ends, never locally.
_Avoid_: AI summary, recap section

**Brief**:
The single-paragraph recap within a Spotlight Summary.

**Output Directory**:
The local folder Call Files are written to. The folder itself is the source of truth for what has been synced.
_Avoid_: Export folder, destination

**High-Water Mark**:
The point in time sync resumes from, derived by scanning the Output Directory for the newest Call File date. Never stored separately. (From the flood line on a riverbank, the highest point the water reached.)
_Avoid_: Watermark, checkpoint, resume point, last-synced date

**Overlap Window**:
The number of days (default 3) sync backs up from the High-Water Mark, so late-arriving Transcripts and Spotlight Summaries are caught on subsequent runs.

**Placeholder**:
The marker text written into a Call File's Summary section when the Spotlight Summary was not yet available. A Call File containing a Placeholder is incomplete and eligible for healing.

**Healing**:
Rewriting an incomplete Call File within the Overlap Window once its Spotlight Summary becomes available.
_Avoid_: Backfill, refresh, repair

**Party**:
A person on a Call, with name, email, title, and an affiliation of Internal, External, or Unknown. Unknown usually means an unmatched external person (phone caller, unrecognized email), not a colleague.
_Avoid_: Attendee, speaker, participant, contact

**Customer Call**:
A Call with at least one Party whose affiliation is not Internal. Defined by exclusion because genuinely external people often carry the Unknown affiliation.
_Avoid_: External call, sales call

**Internal Call**:
A Call where every Party's affiliation is Internal. Excluded from sync.
